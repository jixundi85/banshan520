# 半山AIX平台 - 服务器部署指南

## 一、项目概述

**项目名称**：半山AIX - AI创作者变现与服务平台
**技术栈**：React 18 + Vite + TailwindCSS + React Router
**运行环境**：Node.js 18+ 或 20+

## 二、快速部署（推荐）

### 方法一：使用已构建文件（最简单）

1. **上传文件到服务器**
   ```bash
   # 在服务器上创建目录
   mkdir -p /var/www/aigc-platform

   # 将 dist 目录内容上传到服务器
   scp -r aigc-platform/dist/* user@your-server:/var/www/aigc-platform/
   ```

2. **配置 Nginx**
   ```bash
   # 创建 Nginx 配置文件
   sudo nano /etc/nginx/sites-available/aigc-platform
   ```

   写入以下内容：
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;  # 替换为你的域名或IP

       root /var/www/aigc-platform;
       index index.html;

       # 处理 SPA 路由
       location / {
           try_files $uri $uri/ /index.html;
       }

       # 静态资源缓存
       location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
           expires 30d;
           add_header Cache-Control "public, immutable";
       }

       # 安全头
       add_header X-Frame-Options "SAMEORIGIN" always;
       add_header X-Content-Type-Options "nosniff" always;
       add_header X-XSS-Protection "1; mode=block" always;
   }
   ```

3. **启用站点**
   ```bash
   sudo ln -s /etc/nginx/sites-available/aigc-platform /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl reload nginx
   ```

4. **配置 HTTPS（推荐）**
   ```bash
   sudo apt install certbot python3-certbot-nginx
   sudo certbot --nginx -d your-domain.com
   ```

#### 1. 服务器环境准备


## 五、Nginx 完整配置示例

```nginx
server {
    listen 80;
    listen [::]:80;
    server_name 你的服务器IP或域名;

    root /var/www/aigc-platform;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/javascript application/json application/xml+rss;

    # 日志
    access_log /var/log/nginx/aigc-access.log;
    error_log /var/log/nginx/aigc-error.log;

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # 禁止访问隐藏文件
    location ~ /\. {
        deny all;
    }
}
```

## 六、常见问题

### Q1: 页面空白或404
检查 Nginx 配置中的 `try_files $uri $uri/ /index.html;`

### Q2: 静态资源加载失败
检查 `root` 路径是否正确指向 dist 目录

### Q3: 刷新后404
这是 SPA 路由问题，确保 try_files 配置正确

## 七、一键部署脚本

创建 `deploy.sh`：

```bash
#!/bin/bash

# 配置
DEPLOY_PATH="/var/www/aigc-platform"
SOURCE_PATH="./dist"

echo "===== 半山AIX平台部署脚本 ====="

# 创建目录
sudo mkdir -p $DEPLOY_PATH

# 复制文件
sudo cp -r $SOURCE_PATH/* $DEPLOY_PATH/

# 设置权限
sudo chown -R www-data:www-data $DEPLOY_PATH

echo "部署完成！访问 http://你的服务器IP"
```

运行：
```bash
chmod +x deploy.sh
./deploy.sh
```
